# Security Policy

## Reporting a vulnerability

Do not open a public issue for security-sensitive reports.

Report vulnerabilities privately to the maintainer through GitHub Security Advisories for this repository, if enabled, or by direct maintainer contact before public disclosure.

Include:

- A description of the issue
- Steps to reproduce
- Impact assessment
- Any suggested mitigation

I will aim to acknowledge reports promptly and work toward a fix before public disclosure when reasonably possible.
